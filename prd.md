# 나를 채워주는 다섯 정령 — MVP PRD

---

## 1. 한 줄 설명

사주 기반으로 부족한 오행을 분석해, 그 기운을 상징하는 정령이 먼저 말을 걸어오는 AI 힐링 채팅 서비스.

---

## 2. 대상 사용자

- 번아웃, 무기력, 불안 등 막연한 감정을 느끼는 2030 직장인·대학생
- 사주나 오행에 관심은 있지만 너무 진지한 상담은 부담스러운 사람
- "귀엽고 따뜻한 무언가"와 대화하며 위로받고 싶은 사람

---

## 3. 핵심 기능

### 3-1. 오행 분석
생년월일(사주) 입력 → 오행(목·화·토·금·수) 분포 계산 → 부족한 기운 도출.  
결과를 시각적 그래프(애니메이션)로 보여준다.

**MVP 오행 계산 방식 (단순 공식, Next.js API Route)**

`POST /api/analyze` 에서 처리.  
생년월일의 연·월·일 숫자를 기반으로 각 오행에 점수를 배분.  
정밀한 명리학 계산이 아닌, 경험을 위한 근사치 공식.

```
예: 연도 끝자리 → 오행 매핑
  0, 1 → 金
  2, 3 → 水
  4, 5 → 木
  6, 7 → 火
  8, 9 → 土

월, 일도 같은 방식으로 오행 점수에 누적 합산.
가장 점수가 낮은 오행 = 부족한 기운 = 추천 정령.
```

### 3-2. 정령 추천 + 사용자 선택
오행 분석 결과를 기반으로 부족한 기운이 가장 큰 정령을 동적으로 추천한다.  
추천 텍스트는 상단에 표시되며, 사용자가 카드 5개 중 원하는 정령을 직접 고를 수도 있다.

```
예시 화면 (오행 분석 결과에 따라 추천 정령이 달라짐):
  ┌──────────────────────────────────────────┐
  │ ✨ 오늘 당신에게는 뿌리가 어울려요!      │
  │    (목 기운이 가장 부족합니다)           │
  └──────────────────────────────────────────┘

  [ 뿌리 ]  [ 아로 ]  [ 토토 ]  [ 치링 ]  [ 수수 ]
   ★추천      ·         ·         ·         ·
```

### 3-3. 캐릭터 선택 화면 (카드 5개)
5개 정령 카드를 그리드로 표시.  
각 카드: 정령 이미지 + 이름 + 오행 + 성격 한 줄 소개.  
추천 정령 카드는 시각적으로 강조(뱃지, 테두리 등).

| 오행 | 정령 | 이미지 파일 | 성격 한 줄 |
|------|------|------------|-----------|
| 木 | 뿌리 (Puri) | `1.tree` | 씩씩한 아기 나무, 무기력한 나에게 성장 에너지를 |
| 火 | 아로 (Aro) | `2.fire` | 따뜻한 촛불, 식어버린 열정에 불씨를 |
| 土 | 토토 (Toto) | `3.soil` | 듬직한 찰흙 인형, 불안한 나의 안식처 |
| 水 | 수수 (Susu) | `4.water` | 말랑한 물방울, 번아웃된 나를 흘려보내게 도와주는 상담사 |
| 金 | 치링 (Chiring) | `5.metal` | 반짝이는 보석, 복잡한 머릿속을 정리해주는 가이드 |

> 이미지는 별도 제작 예정. 파일명은 위 기준으로 `public/spirits/` 폴더에 배치.

### 3-4. 채팅 화면 (말풍선 UI)
- 상단 헤더: 뒤로가기 + 정령 이름/이미지
- 중앙: 말풍선 히스토리 (내 말 오른쪽, 정령 말 왼쪽)
- 하단: 입력창 + 전송 버튼 (Enter 전송, Shift+Enter 줄바꿈)
- AI 응답 중: 타이핑 애니메이션 (점 세 개)
- 정령의 첫 인사는 자동으로 먼저 나온다

### 3-5. AI 응답 (OpenRouter API + 스트리밍)
OpenRouter를 통해 AI 모델 호출.  
정령별 `systemPrompt`를 앞에 붙여 캐릭터 말투·세계관 유지.  
응답은 글자 단위로 실시간 출력(SSE 스트리밍).  
**한국어로만 대화**한다.

### 3-6. 감정 감지 인터랙션
대화 중 우울·부정적 단어 감지 시 → 화면 채도 변화 + 정령이 화면을 "톡톡" 건드리는 응원 연출.

### 3-7. 치유 구슬 수집
상담 종료 시 정령이 "오늘 이만큼 채웠어"라며 구슬 아이콘 선물.  
모은 구슬을 프로필에서 볼 수 있는 수집 루프.

## UI 디자인 가이드

### 레퍼런스 스크린샷에서 따올 것
- 상단 중앙에 정령 캐릭터 크게 배치 (floating, 생동감 있게)
- 캐릭터 아래 인사말 텍스트 (크고 친근하게)
- 말풍선 스타일 퀵 액션 버튼들 (채팅 시작 전 대화 유도)
- 채팅창: 입력창이 하단 고정, 심플한 말풍선
- 전체적으로 여백 넉넉하게, 카드 없이 플랫하게

### 색상

| 용도 | 색상 |
|------|------|
| 배경 | 화이트 (`#FFFFFF`) |
| 포인트 컬러 | 따뜻한 살구 / 연한 코랄 |
| 텍스트 | 다크 브라운 계열 (검정 대신) |
| 말풍선 (캐릭터) | 연한 피치 |
| 말풍선 (사용자) | 웜 테라코타 계열 포인트 |
| 버튼 | 연한 살구, 고스트 버튼 |

### 미니멀 원칙
- 그림자 최소화, 배경색 차이로만 섹션 구분
- 아이콘은 선형(outline)
- 카드 경계선 없이 플랫하게
- 폰트: 시스템 폰트 기반, 굵기 차이로 계층 표현

### 화면별 레이아웃

**홈 (오행 분석 전)**
```
[상단] 상태바
[중앙] 정령 캐릭터 (크게, 둥둥 떠있는 느낌)
       인사말: "Hi, OOO님 만나서 정말 반가워요~!"
[하단] 퀵 버튼 2–3개 (말풍선 형태)
       "지금 제일 힘든 게 뭐예요?"
       "오늘 기분은 어때요?"
       입력창 (하단 고정)
```

**결과 화면 (오행 분석 후)**
```
[상단] 오행 그래프 (애니메이션)
       추천 정령 텍스트 배너
[중앙] 정령 카드 5개 그리드
[하단] "OOO와 대화 시작하기" 버튼
```

**채팅 화면**
```
[상단] 뒤로가기 / 정령 이름
[중앙] 말풍선 히스토리 (캐릭터: 연한 피치 / 사용자: 포인트 컬러)
       정령 아바타 (좌상단 작게)
[하단] 입력창 + 전송 버튼
```

## 4. 안 만들 것 (MVP 제외)

- **회원가입 / 로그인** — 비회원 체험만으로 MVP 검증
- **실제 사주 엔진** — MVP는 생년월일을 단순 공식으로 오행 매핑 (정밀도보다 경험 우선)
- **구슬 영구 저장 / 서버 DB** — MVP는 localStorage에 임시 저장
- **다중 정령 동시 대화** — 한 번에 정령 한 명만
- **소셜 공유 / 커뮤니티** — 나중에

---

## 5. 사용자 흐름 (5단계)

```
1. 접속
   └─ 서비스 소개 화면 + "내 정령 찾기" 버튼

2. 사주 입력
   └─ 생년월일 입력 (양력/음력 선택)

3. 오행 분석 결과
   └─ 애니메이션으로 오행 그래프 등장
   └─ 부족한 기운 기반으로 정령 추천 텍스트 표시

4. 정령 선택 (추천 or 직접 선택)
   └─ 카드 5개 중 선택
   └─ 선택한 정령이 화면에 나타나며 먼저 인사

5. 대화 & 치유 구슬 획득
   └─ 감정 인터랙션 포함 채팅
   └─ 종료 시 구슬 수령
```

## 6. 성공 기준

| 기준 | 측정 방법 |
|------|-----------|
| 생년월일 입력 → 정령 추천까지 오류 없이 도달 | 직접 테스트 5회 |
| 추천을 무시하고 다른 정령 선택도 정상 동작 | 직접 테스트 |
| 정령과 3턴 이상 대화가 자연스럽게 이어짐 | 사용자 테스트 |
| "또 하고 싶다"는 반응이 테스터 과반수에서 나옴 | 지인 5명 테스트 |
| 모바일에서도 레이아웃 깨짐 없이 동작 | Chrome DevTools |

## 기술 스택

| 역할 | 선택 |
|------|------|
| 프레임워크 | Next.js (App Router) |
| 스타일 | Tailwind CSS |
| AI | OpenRouter API (스트리밍) |

| 엔드포인트 | 역할 |
|-----------|------|
| `POST /api/analyze` | 생년월일 → 오행 점수 계산 → 추천 정령 반환 |
| `POST /api/chat` | 정령 systemPrompt + 대화 히스토리 → AI 스트리밍 응답 |
